# 总论

1. 除非及其特殊情况.根据知识点去读对应的章节.
2. 通读Windows internals效果非常不好,应当结合用到的知识点去读


## dinvk / puerto (动态调用与底层基石)

这个项目的本质是脱离 Windows 默认的加载与异常处理体系，建立黑客自己的运行时（Runtime）。

| 源码文件 | 核心物理实质 / OS 机制 | WI7 精确映射指南 (第 7 版) | 阅读建议与战术目的 |
| --- | --- | --- | --- |
| module.rs | 读取 TEB 找到 PEB，遍历 PEB_LDR_DATA 链表寻找模块基址；解析 PE 头部的 Export Directory 匹配函数。 | Part 1 - Ch 3 (Process and Jobs)<br>→ The Image Loader (映像加载器) | 必读。精确理解 OS 是如何将 DLL 的物理路径映射为虚拟内存中的 Ldr 结构体的。这是无文件加载的基石。 |
| helper.rs | 纯粹的 PE 文件结构（DOS Header, NT Header）指针偏移计算。 | (本书不侧重 PE 文件格式) 可参考官方 PE/COFF 规范文档。 | 理解偏移量（RVA 到 VA 的转换）。 |
| allocator.rs | 获取默认进程堆 (GetProcessHeap) 或调用 RtlAllocateHeap 进行底层内存操作。 | Part 1 - Ch 5 (Memory Management)<br>→ Heap Manager (堆管理器) | 理解 _HEAP 结构，弄懂为什么在底层免杀中绝对不能越界写堆，以及 Segment Heap 与 LFH 的差异。 |
| breakpoint.rs | 利用硬件断点（Dr0-Dr3 寄存器）结合异常处理劫持执行流（如 VEH 劫持）。 | Part 2 - Ch 8 (System Mechanisms)<br>→ Trap Dispatching -> Hardware Breakpoints | 了解内核是如何捕获硬件断点并分发给用户态的异常处理程序的。用于高级动态调用（规避 API Hook）。 |
| panic.rs | 自定义 #![no_std] 环境下的崩溃处理，接管进程死亡的最后一瞬间。 | Part 2 - Ch 8 (System Mechanisms)<br>→ Exception Dispatching (Unhandled Exceptions) | 理解如果木马不截获 Panic，Windows 的 WER (Windows Error Reporting) 是怎么把崩溃转储发给微软的（极度危险的暴露）。 |
| console.rs | 封装 OutputDebugStringA 进行静默输出。 | Part 2 - Ch 10 (Management...)<br>→ User-Mode Debugging | 了解调试管道是如何将字符串发送给内核的 DbgPrint 缓冲区的。 |
| syscall/asm.rs<br>syscall/x86_64.rs | 纯汇编实现：把 SSN 放入 EAX，把返回地址放入 R10，执行 syscall。 | Part 2 - Ch 8 (System Mechanisms)<br>→ System Service Dispatching | 核心必读。深刻理解 CPU 从 Ring 3 (用户态) 跃迁到 Ring 0 (内核态) 时的 KiSystemCall64 处理流程。 |
| 其余文件 (hash.rs, macros.rs, types.rs, lib.rs) | 纯 Rust 语言逻辑（哈希算法、宏定义、FFI 类型声明），不涉及具体 OS 机制。 | 无直接映射 | 熟悉 Rust 语法和类型布局。 |

## uwd (系统级调用栈欺骗工具库)

此库的功能极其聚焦：欺骗操作系统的栈回溯机制。

| 源码文件 | 核心物理实质 / OS 机制 | WI7 精确映射指南 (第 7 版) | 阅读建议与战术目的 |
| --- | --- | --- | --- |
| uwd.rs | 读取目标 DLL 的异常目录 (.pdata 节)，解析 IMAGE_RUNTIME_FUNCTION 和 UNWIND_INFO，计算每个系统 API 在执行时消耗的精确栈深度（Stack Size）。 | Part 2 - Ch 8 (System Mechanisms)<br>→ Exception Dispatching (重点看 x64 Unwind 机制) | 全书防守规避的最高价值章节。在 x64 中，栈是不能随便乱飞的。EDR 就是靠这套 Unwind 机制查杀，您必须比 EDR 更懂这套回溯规则，才能写出完美的伪造偏移。 |
| util.rs | 搜索特定机器码（如 jmp rbx 等 ROP Gadgets），内存匹配与洗牌混淆。 | Part 1 - Ch 5 (Memory Management)<br>→ Virtual Address Space | 了解 Windows 是如何对齐内存页的（Page Alignment），保证搜索 Gadget 时不触发跨页异常（Page Fault）。 |
| 其余文件 (lib.rs, types.rs) | Rust 接口封装与结构体定义。 | 无直接映射 | - |

## hypnus / samoa (终极执行流混淆与内存隐藏)

这是操作系统的调度器级别的对抗。它将 uwd 的伪造栈技术，与操作系统的异步和多线程机制结合。

| 源码文件 | 核心物理实质 / OS 机制 | WI7 精确映射指南 (第 7 版) | 阅读建议与战术目的 |
| --- | --- | --- | --- |
| hypnus.rs<br>(Timer & Wait 模式) | 调用 TpAllocPool, TpAllocTimer 创建私有线程池。木马主线程休眠，让系统合法的 Worker 线程去执行 ROP 链（加密/解密/触发功能）。 | Part 2 - Ch 8 (System Mechanisms)<br>→ Worker Factories (工作线程工厂) | 理解 Windows 10 新引入的线程池底层对象（Worker Factory）。弄懂为什么 EDR 很难区分恶意 Worker 线程和正常系统 Worker 线程。 |
| hypnus.rs<br>(Foliage 模式) | 创建挂起线程，用 NtQueueApcThread 将 ROP 上下文塞入该线程的 APC 队列，唤醒线程执行。 | Part 2 - Ch 8 (System Mechanisms)<br>→ Asynchronous Procedure Calls (APCs) | 必须死磕的章节。理解什么是 KAPC 和 UAPC，弄懂线程状态（Alertable Wait）对触发 APC 的决定性作用。这是执行流劫持的艺术。 |
| hypnus.rs<br>(上下文操作) | 调用 NtGetContextThread / NtSetContextThread 暴力覆写 CPU 寄存器。 | Part 1 - Ch 4 (Threads)<br>→ Thread Internals (线程内部机制) | 理解线程切换（Context Switch）的本质。知道 CONTEXT 结构体是如何代表一个线程的"灵魂"的。 |
| spoof.rs | 将 hypnus 的 ROP 链指令，具体落地到内存指针的加减和覆盖（如劫持 RIP, 伪造 RSP, RBP 返回地址）。 | Part 2 - Ch 8 (System Mechanisms)<br>→ Exception Dispatching (Stack Unwinding) | 与 uwd.rs 相互印证。uwd 负责计算出深度，spoof.rs 负责把这些深度在物理内存上"造假"拼装起来。 |
| cfg.rs | 针对开启了 CFG（控制流保护）的进程，调用 SetProcessValidCallTargets 强行将恶意的跳转地址标记为合法，避免被内核击杀。 | Part 1 - Ch 7 (Security)<br>→ Exploit Mitigation -> Control Flow Guard | 了解 Windows 是怎么用内存中一张巨大的 Bitmap（位图）来验证每一个间接调用的。这是突破现代 Win11 漏洞利用缓解措施的必修课。 |
| config.rs | 统一的动态地址解析注册中心；调用 NtProtectVirtualMemory 修改内存属性（RX -> RW -> RX）。 | Part 1 - Ch 5 (Memory Management)<br>→ Page Fault Handling / Memory Protection | 理解 DEP (数据执行保护) 和内存页权限。知道为什么写 Shellcode 时如果不开 RWX/RX 会触发蓝屏级别的异常。 |
| gadget.rs | 从加载的 DLL 内存中挖掘 ROP/JOP 指令跳板。 | Part 1 - Ch 5 (Memory Management)<br>→ System Memory Allocations | 了解 DLL 在物理内存中究竟是怎么排布的。 |
| allocator.rs | （复用之前分析）建立私有堆，便于 hypnus 在主线程休眠时进行堆内存的全量异或加密。 | Part 1 - Ch 5 (Memory Management)<br>→ Heap Manager | 理解 RtlWalkHeap 遍历堆结构的底层逻辑。 |
