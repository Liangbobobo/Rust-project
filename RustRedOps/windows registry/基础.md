关于Windows Registry  
在win3.x时代,win依赖上万个.ini文件,这导致了ini的混乱.随后引入了注册表的解决方案

1. 分层树状数据库:将所有硬件驱动配置,用户偏好,网络设置,安装信息统一存储在一个结构化的树状数据库中
2. Hive配置单元物理存储:注册表在磁盘上切分为几个大的二进制文件(Hives).如SAM,SECURITY,SOFTWARE,SYSTEM,DEFAULT
3. 内核级服务:由内核中的配置管理器直接管理.配置的读取是纳秒级的

## Google Project Zero:The Windows Registry Adventure#2: A brief history of the feature

注册表:  
1. In essence, the registry is a hierarchical分等级的 database made of named "keys" and "values", used by Windows and applications to store a variety of settings and configuration data.
2. It is represented by a tree structure, in which keys may have one or more sub-keys, and every subkey is associated with exactly one parent key.
3. Furthermore, every key may also contain one or more values, which have a type (integer, string, binary blob etc.) and are used to store actual data in the registry.
4. Every key can be uniquely identified by its name and the names of all of its ascendants separated by the special backslash character ('\'), and starting with the name of one of the top-level keys (HKEY_LOCAL_MACHINE, HKEY_USERS, etc.).For example, a full registry path may look like this: HKEY_CURRENT_USER\Software\Microsoft\Windows.
5. At a high level, this closely resembles类似 the structure of a file system, where the top-level key is equivalent to the root of a mounted disk partition挂载的磁盘分区 (e.g. `C:\`), keys are equivalent to directories, and values are equivalent to files. One important distinction, however, is that keys are the only type of securable objects in the registry, and values play a much lesser role in the database than files do in the file system.
6. Furthermore, specific subtrees of the registry are stored on disk in binary files called registry hives, and the hive mount points don't necessarily correspond符合 one-to-one to the top-level keys (e.g. the C:\Windows\system32\config\SOFTWARE hive is mounted under HKEY_LOCAL_MACHINE\Software, a one-level nested key).

Fundamentally, there are only a few basic operations that can be performed in the registry. These operations are summarized in the table below:
| Hives: |  |  |
| :--- | :--- | :--- |
| Load hive | Unload hive | Flush hive to disk |
| Keys: |  |  |
| Open key | Create key | Delete key |
| Rename key | Set/query key security | Set/query key flags |
| Enumerate subkeys | Notify on key change | Query key path |
| Query number of open subkeys | Close key handle |  |
| Values: 
| Set value | Delete value | Enumerate values |
| Query value data |  |  |
