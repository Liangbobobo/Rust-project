
# NAT网络-家用路由器/手机热点

由于CGNAT (Carrier-Grade NAT，大内网/运营商级NAT)的存在,在中国/印度等IPv4地址短缺的国家,家用路由器并不直接分配公网IPv4地址,拿到的是一个共享内网 IP(100.64.0.0/10或10.x.x.x).  
isp将很多家用路由器再次映射到一个真正的全球唯一的公网IPv4上,这种情况下整个小区甚至整个区公用这一个IPv4地址

这种情况下:  
得到的IPv4地址可能是第三层的cgnat的共享IP,不能确定后面的路由器,更不能确定路由器后面的个人电脑.此时如果硬闯,可能会渗透到运营商骨干网络设备,这是及其危险的操作.

在IPv4的cgnat网络环境下, 从外部硬闯几乎不可能.只要运营商网关（LTS）不主动为你做映射，你的数据包就根本无法到达家用路由器的 WAN 口  
在这种环境下,应关注:  
1. IPv6
2. upnp映射端口的情况
3. 换成诱导点击方案

其他及其困难的突破口:  
1. 针对运营商边界网关的“中间人”劫持 (BGP/ISP Level Hijacking)
2. 基于“合法中间协议”的信道劫持 (App-Level Side Channel)
3. 利用 NAT 映射规律进行“盲打”注入 (NAT Blind Injection).主要用到TCP序列号盲猜,

# NAT网络-

## 检测目标处于哪种网络环境

### 路由追踪

traceroute命令,检查网络链路

### 端口转发测试 (Port Forwarding Probe)

CGNAT 下，成百上千个用户共享一个公网 IP.如果发现一个 IP开着非常杂乱的、不符合逻辑的服务（例如：同时开着海康威视监控、群晖 NAS 登录口、还有个 Windows RDP）.这大概率不是一个人的设备，而是该公网 IP 背后多个用户通过 UPnP映射出来的服务总和。这就是 CGNAT 的典型指纹

###  基于协议行为的探测 (NAT Type Detection)

利用 STUN (Session Traversal Utilities for NAT) 协议进行探测。这是 P2P软件（如 BitTorrent, 游戏联机）常用的技术

### DNS 记录反向查找 (Reverse DNS)

`nslookup [目标IP]`   
某些运营商会为 CGNAT 的公网出口 IP 绑定特定的 PTR 记录(例如：cgnat-pool-xxx.isp.com),
