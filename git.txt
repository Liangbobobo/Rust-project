关于换行符:
git config --global core.autocrlf true
提交时: 自动将 CRLF 转换为 LF

注意时本地未同步的情况,不是本地上传时候的错误
本地未同步的情况:
git pull --rebase origin main
error: cannot pull with rebase: You have unstaged changes.
error: please commit or stash them.

解决:  
1.丢弃 (Reset) 您的本地修改

git reset --hard HEAD

2. 储藏 (Stash) 您的修改 (推荐)

git stash
git pull --rebase origin main
git stash pop

3. 提交 (Commit) 您的修改:



git add .
// 重置暂存区 git reset
git commit -m "初始化时的项目"
git push -u origin main

更新本地仓库与github仓库保持一致
git pull --rebase origin main

git add .  向这个本地的代码仓库添加当前目录的所有改动

git add -p  只添加某个文件

git status  查看仓库状态 可以查看改便的文件状态

git commit -m "2021.2.5初始使用git整理的文档"

git push origin master  推送到远程仓库 


重要：
git pull --rebase origin master  将远程库与本地库同步
